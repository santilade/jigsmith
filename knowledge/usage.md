# Rubric — Usage

The what/when/how-much lens. Mostly descriptive; few forge candidates come from
here, but it frames everything else. Master-tended — update as your read sharpens.

## Read for
- **Cross-agent split (`by_agent`).** Which agent for which job? Deep multi-tool
  sessions vs quick one-shots. Divergence is a *finding to surface*, not a forge —
  unless one agent is being used against its grain (e.g. heavy edits in a tool you
  mostly use for Q&A).
- **Model routing.** Opus on trivial edits, or a weak model on hard tasks, is
  waste. Cadence of downshifting (or never) is a habit signal.
- **Token shape.** Cache-read vs input ratio hints at context reuse; huge input
  per turn points at context bloat (hand off to the Context lens).
- **Temporal.** Bursty vs steady; busiest hours/days. Mostly color, not a jig.

## Flag (don't over-forge)
- Usage stats rarely justify a tool on their own. Resist forging "a dashboard for
  X" — that's what the Profile already is.

## Payback framing
- Per-instance cost here is usually small; the value is *pointing the other lenses*
  at the right project/agent/time.
